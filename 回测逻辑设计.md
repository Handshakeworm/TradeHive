# 回测系统设计

## 核心思路

分两阶段执行，解耦决策生成和绩效评估：

```
阶段1: 批量决策 —— 遍历日期，调用 propagate()，保存每天的结构化交易指令到 JSON
阶段2: 模拟回测 —— 读取所有指令 + 拉取真实行情，按指令模拟下单，计算绩效
```

这样做的好处：
- 阶段1 是昂贵的（LLM 调用），跑一次就够
- 阶段2 是廉价的（纯计算），可以反复调参数重跑
- 中间结果持久化，随时可以从断点恢复

---

## 前提: Agent 结构化输出

Portfolio Manager 需要输出结构化的交易指令，而非自由文本。结构化输出应包含：

```json
{
  "rating": "BUY",
  "target_position_pct": 0.05,
  "entry_plan": [
    {
      "phase": "A",
      "action": "BUY",
      "allocation_pct": 0.30,
      "trigger": "IMMEDIATE",
      "limit_price": null
    },
    {
      "phase": "B",
      "action": "BUY",
      "allocation_pct": 0.35,
      "trigger": "PRICE_BELOW",
      "limit_price": 86.30
    },
    {
      "phase": "C",
      "action": "BUY",
      "allocation_pct": 0.35,
      "trigger": "PRICE_ABOVE",
      "limit_price": 95.00
    }
  ],
  "stop_loss": 82.00,
  "take_profit": 100.00,
  "time_horizon_days": 30
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `rating` | enum | BUY / OVERWEIGHT / HOLD / UNDERWEIGHT / SELL |
| `target_position_pct` | float | 目标仓位占组合比例，如 0.05 = 组合总市值的 5% |
| `entry_plan` | list | 分批建仓/减仓计划 |
| `entry_plan[].phase` | str | 批次标识 |
| `entry_plan[].action` | enum | BUY / SELL |
| `entry_plan[].allocation_pct` | float | 该批次占**目标仓位**的比例（所有批次之和应 = 1.0） |
| `entry_plan[].trigger` | enum | IMMEDIATE / PRICE_BELOW / PRICE_ABOVE |
| `entry_plan[].limit_price` | float? | 触发价格，IMMEDIATE 时为 null |
| `stop_loss` | float? | 止损价，null 表示不设止损 |
| `take_profit` | float? | 止盈价，null 表示不设止盈 |
| `time_horizon_days` | int | 本次决策的有效期（天数），超期未触发的挂单自动取消 |

`allocation_pct` 计算示例：

```
假设组合总市值 = $100,000，target_position_pct = 0.05
→ 目标仓位 = $100,000 × 5% = $5,000

Phase A (allocation_pct = 0.30): 买入 $5,000 × 30% = $1,500
Phase B (allocation_pct = 0.35): 买入 $5,000 × 35% = $1,750
Phase C (allocation_pct = 0.35): 买入 $5,000 × 35% = $1,750
                                              合计 = $5,000 (100%)
```

即：`allocation_pct` 不是占组合的比例，而是占目标仓位的比例。三个批次加起来 = 100% 的目标仓位。

## 阶段1: 批量决策生成

### 输入
```python
ticker = "NVDA"
start_date = "2024-01-02"    # 回测起始日
end_date = "2024-06-28"      # 回测结束日
decision_interval = 5        # 每隔几个交易日做一次决策（默认5=周频）
```

### 流程
```
1. 用 yfinance 拉取 start_date ~ end_date 的所有交易日列表
2. 按 decision_interval 采样出决策日期列表
3. 对每个决策日期:
   a. 检查 eval_results/ 下是否已有该日期的结果（支持断点续跑）
   b. 如果没有，调用 ta.propagate(ticker, date)
   c. 从结构化输出中提取交易指令
   d. 校验字段合法性（rating 枚举、价格为正数等）
   e. 保存到统一的决策记录文件
```

### 决策记录格式

文件: `eval_results/{ticker}/backtest_decisions.json`

```json
{
  "ticker": "NVDA",
  "config": {
    "start_date": "2024-01-02",
    "end_date": "2024-06-28",
    "decision_interval": 5,
    "llm_model": "gpt-5-mini"
  },
  "decisions": [
    {
      "date": "2024-01-02",
      "rating": "BUY",
      "target_position_pct": 0.05,
      "entry_plan": [
        {"phase": "A", "action": "BUY", "allocation_pct": 0.30, "trigger": "IMMEDIATE", "limit_price": null},
        {"phase": "B", "action": "BUY", "allocation_pct": 0.35, "trigger": "PRICE_BELOW", "limit_price": 86.30},
        {"phase": "C", "action": "BUY", "allocation_pct": 0.35, "trigger": "PRICE_ABOVE", "limit_price": 95.00}
      ],
      "stop_loss": 82.00,
      "take_profit": 100.00,
      "time_horizon_days": 30
    },
    {
      "date": "2024-01-08",
      "rating": "HOLD",
      "target_position_pct": null,
      "entry_plan": [],
      "stop_loss": null,
      "take_profit": null,
      "time_horizon_days": null
    }
  ]
}
```

## 阶段2: 模拟回测

阶段1 跑完所有决策日期后，阶段2 一次性拉取完整行情数据，逐日比对决策执行情况。

### 模拟规则

```
- 执行价格: 决策日的下一个交易日的开盘价
- 交易成本: 可配置，默认单边 0.1%（滑点+手续费）
- 初始资金: 可配置，默认 $100,000
- 仅做多，不支持杠杆和做空
```

### 模拟流程

```
输入:
  backtest_decisions.json (阶段1产出)
  完整行情数据 (yfinance 一次性拉取)

初始状态:
  cash = 100000
  position = 0 (持仓股数)
  trades = []  (交易记录)

遍历每个决策:
  1. 读取该决策的 rating 和 target_position_pct
  2. 取下一个交易日的开盘价作为执行价
  3. 计算目标持仓:
     target_value = 当前组合总市值 * target_position_pct
     target_shares = floor(target_value / 执行价)
  4. 计算差额 = target_shares - position
     - 差额 > 0: 买入，cash 减少，position 增加
     - 差额 < 0: 卖出，cash 增加，position 减少
     - 差额 = 0 或 HOLD: 不操作
  5. 扣除交易成本
  6. 记录这笔交易

遍历结束后:
  用完整行情数据回填每日组合市值 = cash + position * 当日收盘价
```

### HOLD 的处理

HOLD 表示不操作，维持当前仓位不变。不需要计算目标仓位。

### 止损/止盈检查（简化版）

不做逐日盘中检查，只在**两次决策之间**简单判断：

```
如果决策设置了 stop_loss:
  在下一次决策到来之前，如果某天收盘价 <= stop_loss → 次日开盘价清仓
如果决策设置了 take_profit:
  在下一次决策到来之前，如果某天收盘价 >= take_profit → 次日开盘价清仓
```

### 基准对比

同时计算 **Buy & Hold** 基准：
```
第一个交易日以开盘价全仓买入，持有到最后一天
基准净值曲线 = 初始资金 * (每日收盘价 / 首日开盘价)
```

## 绩效指标

### 核心指标

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 年化收益率 (CAGR) | 测试期间的复合年增长率 | (期末市值 / 初始资金) ^ (252 / 交易天数) - 1 |
| 波动率 (Volatility) | 收益率的标准差，衡量收益的不确定性 | 日收益率标准差 * sqrt(252) |
| 夏普比率 (Sharpe Ratio) | 风险调整后的收益，越高越好 | (年化收益 - 无风险利率) / 年化波动率，无风险利率默认 4% |
| 最大回撤 (Max Drawdown) | 从历史最高点到最低点的最大跌幅 | max((峰值 - 谷值) / 峰值) |
| 换手率 (Turnover) | 持仓变化的频率和幅度 | 每次调仓的交易金额 / 调仓时组合总市值，取所有调仓的平均值 |
| 交易成本 (Transaction Costs) | 交易带来的总成本影响 | 所有交易的手续费 + 滑点累计 |

### 计算细节

**年化收益率 (CAGR)**
```python
total_return = (final_value - initial_cash) / initial_cash
cagr = (1 + total_return) ** (252 / total_trading_days) - 1
```

**波动率 (Volatility)**
```python
daily_returns = portfolio_values.pct_change().dropna()
volatility = daily_returns.std() * sqrt(252)
```

**夏普比率 (Sharpe Ratio)**
```python
risk_free_rate = 0.04
sharpe = (cagr - risk_free_rate) / volatility
```

**最大回撤 (Max Drawdown)**
```python
peak = portfolio_values.cummax()
drawdown = (portfolio_values - peak) / peak
max_drawdown = drawdown.min()  # 负值，如 -0.15 表示 15% 回撤
```

**换手率 (Turnover)**
```python
# 每次调仓时:
turnover_i = abs(交易金额) / 调仓时组合总市值
# 汇总:
avg_turnover = mean(所有 turnover_i)             # 平均单次换手率
total_turnover = sum(所有 turnover_i)             # 累计换手率
annualized_turnover = total_turnover * (252 / 交易天数)  # 年化换手率
```

**交易成本 (Transaction Costs)**
```python
# 每笔交易:
cost_i = abs(交易股数) * 执行价 * commission_rate
# 汇总:
total_cost = sum(所有 cost_i)
cost_drag = total_cost / initial_cash  # 成本拖累比例
```

所有指标同时计算策略和 Buy & Hold 基准的值，方便对比。

## 输出

### 文件输出

```
eval_results/{ticker}/backtest_results/
├── backtest_decisions.json      # 阶段1: 每日结构化决策
├── backtest_trades.json         # 交易记录 (日期、方向、数量、价格、盈亏)
├── backtest_daily.csv           # 每日净值 (日期、市值、持仓、现金、基准市值)
├── backtest_metrics.json        # 绩效指标汇总
└── backtest_report.md           # 可读的回测报告
```

## 代码结构

```
tradingagents/
└── backtest/
    ├── __init__.py
    ├── runner.py           # 阶段1: 批量决策循环, 断点续跑
    ├── simulator.py        # 阶段2: 模拟交易 + 绩效计算
    └── report.py           # 生成报告
```

### 使用方式

```python
from tradingagents.backtest.runner import BacktestRunner
from tradingagents.backtest.simulator import TradeSimulator

# 阶段1: 批量生成决策 (耗时, LLM调用)
runner = BacktestRunner(config=config)
runner.run(
    ticker="NVDA",
    start_date="2024-01-02",
    end_date="2024-06-28",
    interval=5,
)

# 阶段2: 模拟回测 (快速, 纯计算, 可反复调参)
simulator = TradeSimulator(
    ticker="NVDA",
    initial_cash=100000,
    commission_rate=0.001,
)
results = simulator.run(decisions_path="eval_results/NVDA/backtest_results/backtest_decisions.json")
results.print_metrics()
results.save_report("eval_results/NVDA/backtest_results/")
```
